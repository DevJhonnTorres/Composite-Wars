¿QUÉ VAS A APRENDER?

El patrón Composite permite tratar objetos individuales y grupos de objetos de manera uniforme, construyendo estructuras jerárquicas tipo árbol. Leaf y Composite implementan la misma interfaz — el cliente no distingue entre ellos.

IMPLEMENTACIÓN COMPLETA EN JAVA — el código que ejecuta el juego
// 1. COMPONENT — Interfaz común
interface Unidad {
    void atacar();   // ← misma firma
    void mover();    //    para TODOS
    void defender(); //    los objetos
}
// 2. LEAF — Objeto básico sin hijos
class Soldado implements Unidad {
    private int poder = 20;

    @Override
    public void atacar() {
        // actúa solo — no delega
        print("Soldado: " + poder + " dmg");
    }
    // mover() y defender() igual...
}
// 3. COMPOSITE — Contiene otros Unidad
class Grupo implements Unidad {
    List<Unidad> hijos = new ArrayList<>();

    public void agregar(Unidad u)  { hijos.add(u); }
    public void eliminar(Unidad u) { hijos.remove(u); }

    @Override
    public void atacar() {
        // DELEGACIÓN RECURSIVA ←←←
        for (Unidad hijo : hijos)
            hijo.atacar(); // Leaf o Composite
    }
}
// 4. CLIENTE — Usa la interfaz Unidad
Unidad s1  = new Soldado("S1"); // Leaf
Unidad med = new Medico("M1");  // Leaf

Grupo alpha = new Grupo("Alpha");
alpha.agregar(s1);   // añadir Leaf
alpha.agregar(med);  // añadir Leaf

Grupo ejercito = new Grupo("Ejercito");
ejercito.agregar(alpha);         // Composite
ejercito.agregar(new Tanque());  // Leaf

// ¡Misma línea de código para ambos!
s1.atacar();        // Leaf:      1 respuesta
alpha.atacar();     // Composite: 2 respuestas
ejercito.atacar();  // Composite: 3 respuestas
FASE 01 LEAF — OBJETOS INDIVIDUALES

CONCEPTO: El Leaf es la unidad más básica. No puede contener otros objetos. Actúa de forma completamente independiente. Implementa la interfaz directamente.

EN EL JUEGO: Arrastra un soldado al árbol. Ejecuta ATACAR. Observa cómo una sola unidad responde.

🪖 Soldado — 20 dmg  |  🚑 Médico — 30 heal  |  🚓 Tanque — 50 dmg
// Leaf.java
class Soldado implements Unidad {
  public void atacar() {
    // Solo actúa este objeto
    print("Soldado: 20 dmg");
  }
}
⬡ Sin hijos — actúa solo
FASE 02 COMPOSITE — GRUPOS

CONCEPTO: El Composite contiene múltiples objetos. Cuando ejecutas una acción, la delega automáticamente a todos sus hijos. El cliente lo usa igual que un Leaf.

EN EL JUEGO: El campo ya tiene un grupo. Arrastra ≥2 unidades dentro. Ejecuta ATACAR — todos responden a la vez.

// Composite.java
class Escuadron implements Unidad {
  List<Unidad> hijos;

  public void atacar() {
    for (Unidad u : hijos)
      u.atacar(); // ← delega
  }
}
⬡ Contiene otros objetos
FASE 03 ÁRBOL — JERARQUÍAS ANIDADAS

CONCEPTO: Los Composite pueden contener otros Composite, formando un árbol. Cada nivel puede tener hijos de cualquier tipo — Leaf o Composite — sin restricciones.

EN EL JUEGO: Usa "📦 Crear Grupo" para anidar grupos dentro de grupos. Observa el árbol jerárquico que se construye en tiempo real.

FASE 04 RECURSIVIDAD — UN SOLO COMANDO

CONCEPTO: Con UNA SOLA llamada en la raíz, la acción se propaga recursivamente por toda la jerarquía. Leaf y Composite responden idénticamente — eso es el poder del patrón.

EN EL JUEGO: Construye un ejército con ≥4 unidades anidadas. Ejecuta ATACAR UNA VEZ — observa todos los resultados en el log.
